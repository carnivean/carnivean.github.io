Assignment 1

2DSmoothArray - Noch einfach

  • Wenn ihr malloc benutzen wollt für das Array, müsst Ihr #include "stdlib.h" hinzufügen
  • smoothArray kann zum Beispiel über eine 4-fach verschachtelte Loops gelöst werden
  • smoothArray stellt sicher, dass eure Indizes für x und y > 0 aber x < width und y < height sind
  • rand() gibt einen int zurück, ihr müsst den Wert in einen float verwandeln um zufällige Werte zwischen 0 und 1 zu erhalten

Vectorsort - Mittel

  • Ihr müsst mit großer Wahrscheinlichkeit folgendes einfügen, um die LIbs zu importieren:
#include <vector>
#include <iostream>
#include <algorithm>
  • Um die neue Zahl hinten an den Vektor anzuhängen, könnt ihr vec.push_back(input); nutzen
  • Zum Sortieren müsst ihr einerseits eine Funktion deklarieren, die Werte miteinander vergleicht, und dann std::sort benutzen, um zB aufsteigend zu sortieren:
bool sort_asc(int i, int j) { return (i < j);
// und später im Code
std::sort(vec.begin(), vec.end(), sort_asc);

ConfigurationParser - Mittel

  • Den ConfigParser werdet ihr den Rest des Praktikums verwenden, also nimmt euch ein bisschen Zeit für ihn
  • Eine Datei öffnen könnt ihr mit Hilfe von:
  ifstream fs;
    fs.open(file);
    if (!fs.is_open()) {
        cout << "Wrong Filepath" << endl;
        return;
    }
  • Zur Ausgabe der Werte könnt ihr folgende Methode verwenden:
void ConfigParser::toStdOut()
{
    std::cout << "Spinning: " << this->spinning << std::endl;
    std::cout << "spinSpeed: " << this->spinSpeed << std::endl;
    std::cout << "bg Color: " << this->backgroundColor.r << ", " << this->backgroundColor.b << ", " << this->backgroundColor.g << std::endl;
    std::cout << "terrainWidth: " << this->terrainWidth << std::endl;
    std::cout << "terrainDepth: " << this->terrainDepth << std::endl;
    std::cout << "terrainHeight: " << this->terrainHeight << std::endl;
}

Command Line and Test Heightfield - Mittel

  • mit generator.seed((unsigned long)time(nullptr)); könnt ihr dem Generator die aktuellen Zeit als Startwert geben um zufällige Ergebnisse zu bekommen
  • Eure main Methode muss wie folgt aussehen int _tmain(int argc, _TCHAR* argv[]), wenn ihr nicht _tmain verwendet, wird das Parsen der Argumente nicht funktionieren !
  • Damit ihr das Programm mit Command Line ARguments startet, dann müsst ihr, Rechtsklick auf das Projekt –> Debugging –> Command Line Arguments und dort zB folgendes hinschreiben -r 8 -o_height terrain_height.png -o_color terrain_color.png -o_normal terrain_normal.png (der Wert für die Resolution wird nachhe rnatürlich höher sein)
  • Überprüft zuerst die gesammte Anzahl an Argumente, ob diese stimmt und danach die einzelnen Argumente
  • Die Überprüfung der Argumente könnte wie folgt aussehen: !_tcscmp(argv[1], _T("-r"))
  • Die Konvertierung der Eingaben geht wie folgt: _tstoi(argv[2])
  • Ihr müsst zuerst ein std::wstring aus den Parametern erzeugen und diesen dann in einen “normalen” std::string konvertieren, ein kleines Beispiel:
  std::wstring temp_normal(argv[8]);
  std::string pNormal = std::string(temp_normal.begin(), temp_normal.end());
  • Für die normalverteilten Zufallswerte müsst ihr erst die Mitte und Standardabweichung berechnen und dann: std::normal_distribution<float> distribution(midPoint, stdDev);

Diamond-Square-Algorithmus - Es wird anspruchsvoll

  • Übungen und zusätzliche Informationen (speziell zur Klausurvorbereitung) über den Algorithmus könnt ihr unter /algorithms/dsa finden
  • Ich würde empfehlen, für den Diamond und Square Step jeweils eigene Funktionen zu machen, um es übersichtlicher zu machen
  • Für den DSA Algorithmus braucht ihr ein Vector der Größe (Resolution + 1)*(Resolution + 1), also für ein Heightfield der Größe 4x4 ein Vector der Größe 5x5
  • Nachdem der DSA Algorithmus durchgelaufen ist, übernimmt die Werte im Bereich (Resolution)*(Resolution) und speichert dies
