  • Error-Message with the getter-function for struct Color backgroundColor - Ungültiger Verweis auf nicht-statischen Member

    Richtig wäre: ConfigParser::Color ConfigParser::getBackgroundColor() { return backgroundColor; }

    Das ist so eine Sache mit dem Scope Operator :: (Tipp: Benutz Ctrl+Space um zu sehen, was gerade sichtbar ist)

    Siehe auch https://msdn.microsoft.com/en-us/library/b451xz31.aspx

    In deinem Fall sagt der Compiler:

    ConfigParser::backgroundColor wird als statische Member Variable interpretiert,

    die aber nicht deklariert wurde in der Klasse. (also als static Color backgroundColor)

    D.h in einer Methode { … } werden alle Methoden und Variablen einer Klasse als statische Member interpretiert wenn sie mit dem Scope Operator verknüpft werden: Class::staticVariable / Class::staticMethod()

    Der Scope Operator gibt an sich an, wo Expressions deklariert wurden. Und je nach Kontext wird er dann entsprechend verwendet um die Sichtbarkeiten aufzulösen.

    In deinem Fall musst du für den Compiler auflösen, wo das Struct Color deklariert wurde (eben in ConfigParser) und wo die Methode getBackgroundColor deklariert wurde (auch in ConfigParser)

    Die Variable backgroundColor ist in der Methode selber von der Klasse ConfigParser sichtbar, d.h dort braucht es dann keine Sichtbarkeitsauflösung und auch keinen Scope Operator.

    Ansonsten wird er auch verwendet in Verbindung mit namespaces. Z.b. sind die template libraries im std namespace deklariert.

    D.h dort braucht es dann zur Verwendung std::templateClass.

    Namespaces sind wichtig, um Zweideutigkeiten und Mehrfachdefinition zu vermeiden bzw. das Programm besser zu strukturieren.

  • GenerateAndStoreImages - Unzulässiger Aufruf einer nicht statischen Memberfunktion

    You cannot call a non-static member function as a static function from outside the class. Instanciate an object first:

    GEDUtils::TextureGenerator gen(...); gen.generateAndStoreImages(...)

    Genauer:

    There is no default constructor. Use our provided constructor instead:

            TextureGenerator(   const std::wstring& texturePathLowFlat,
                            const std::wstring& texturePathLowSteep,
                            const std::wstring& texturePathHighFlat,
                            const std::wstring& texturePathHighSteep
        );

    Please see the header file for more information.

  • LNK1104: cannot open file GEDUtilsd.lib

    Die GEDUtils(d).lib befindet sich in external/Tools/lib/VS2013/win32 für Visual Studio 2013 und in external/Tools/lib/VS2012/win32 für VS2012. Das heißt: 1) Geh auf das Projekt Game -> Rechtsklick -> Properties -> Linker -> General -> Additional Library Directories 2) Ändere dort den bestehenden Pfad (die as am Ende nat+rlich nicht schreiben) ........\external\Tools\lib\$(Platform)\a in ........\external\Tools\lib\VS2013\$(Platform)\a Für VS 2012 dann: ........\external\Tools\lib\VS2012\$(Platform)\a )

    Achtung: Die Einstellungen auch für den Release Mode machen, also bei Configuration auch Release oder All Configurations auswählen!

    Falls der Linker die lib immer noch nicht findet, dann stimmt was mit den Relativen Pfaden ..\a nicht.

  • std::wstring erzeugt memoryleak

    Das Problem ist, dass der wstring an der Stelle, an der du _CrtDumpMemoryLeaks() aufrufst, noch nicht out-of-scope ist und damit noch Speicher belegt.

    Stattdessen solltest du zu Beginn deiner Anwendung folgendes aufrufen:

    _CrtSetDbgFlag ( _CRTDBG_ALLOC_MEM_DF | _CRTDBG_LEAK_CHECK_DF ); (siehe http://msdn.microsoft.com/de-de/library/x98tx3cf.aspx ), dann erfolgt die Ausgabe automatisch NACH Beendigung der main()-Methode.

  • How to calculate the normal vectors

    Compute the normals by using the formulae on slide 3(14). The central differences are explained on the same slide as well. For understanding them, just keep in mind that the derivative is obtained by the slope triangle.

    Normalizing means scaling the vector to unit length, i.e., multiplying the vector by the reciprocal of its length.

    There are two was calcing the normals:

    You calc the x and y vector (which are shown in the picture on the slide) with central differences and than the normal is the cross product of x and y

    You calc the x and y value of the normal by central differences and multiply it by -1 and the z component than is 1 (this is the formular which you can see on the lower right on the slides)

    Zusätzliche Kommentare: - The z in [0,1] just mens that the height field consists of values between 0 and 1 (as generated by the DiamondSquare algorithm, not by random). - You have to bring x,y and z in the same unit space, so multiply x and y by the resolution before normalization ;)
